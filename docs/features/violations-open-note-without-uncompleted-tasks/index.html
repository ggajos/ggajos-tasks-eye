Feature

Violation: open note has no uncompleted tasks

An open note is invalid when it has no unchecked task left to act on.

Rationale

Why it exists.

Why This Violation Exists

Open notes are supposed to represent actionable work. If an open note has no unchecked task, it is not clear what the next action is.

Inbox flags that situation so the user can add a task, close the note, or move it out of active planning.

Executable documentation

Acceptance criteria.

  • status: open notes with no unchecked tasks are reported in Inbox.
  • Missing or blank status is treated as Open for this rule.
  • Open notes with at least one unchecked task do not trigger this violation.
