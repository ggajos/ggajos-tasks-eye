Feature

Done view

The Done view collects completed Tasks items for a selected date and groups them by context and note.

Rationale

Why it exists.

Why Done Exists

Completed work is useful after the checkbox changes. The Done view gives that history a focused surface where the user can review a day, switch dates, and see the work grouped by the same contexts used on the boards.

The view reads completion dates from Tasks metadata so it does not need a separate archive or log.

Executable documentation

Acceptance criteria.

  • Completed tasks are selected by Tasks completion date.
  • Tasks are grouped by folder-derived context and note name.
  • The date picker and previous/next/today controls change the reviewed date.
  • The context filter narrows completed tasks the same way it narrows board rows.
