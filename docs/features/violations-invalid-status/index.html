Violation: invalid status

A note with explicit status outside open, hold, closed, or archived is invalid.

Rationale

Why it exists.

Why This Violation Exists

Status controls which workflow a note belongs to. A typo like waiting should not silently remove the note from the expected board.

Inbox reports invalid explicit statuses while still allowing missing or blank status to use the documented Open default.

Executable documentation

Acceptance criteria.

  • Explicit unsupported status values are reported in Inbox.
  • Non-string status values are reported in Inbox.
  • Missing or blank status does not trigger this violation because it defaults to Open.
