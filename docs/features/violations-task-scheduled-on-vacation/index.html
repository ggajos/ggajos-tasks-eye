Feature

Violation: task scheduled on vacation

An unfinished task with a due date on a configured unavailable day is reported as an Inbox violation.

Rationale

Why it exists.

Why This Violation Exists

Vacation markers show unavailable days, but a scheduled task on one of those days needs stronger treatment. It is not just calendar context; it is a task that should be moved.

Inbox reports the conflict so the user can repair the due date before relying on the Open board.

Executable documentation

Acceptance criteria.

  • Unfinished tasks due on configured vacation dates are reported in Inbox.
  • The violation includes the due date and vacation reason.
  • Rows show only the earliest vacation collision for the note.
  • Tasks on normal working days do not trigger this violation.
