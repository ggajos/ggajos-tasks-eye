Obsidian plugin

Task boards that stay inside your notes.

Tasks Eye turns regular Markdown notes into focused Open, Inbox, Hold, and Done views. It keeps task state in frontmatter and the Obsidian Tasks emoji format, so the vault remains readable without the plugin.

Design intent

A quiet command center for note-centered work.

Tasks Eye is built for a vault where each piece of work lives as a note under Db/. The plugin reads note frontmatter, unfinished Tasks items, due dates, contexts from folders, vacation configuration, and completed-task markers. The result is a small set of native Obsidian views that help decide what to do now, what needs cleanup, and what can wait.

Workflow

From active work to daily review.

  1. Open Review actionable notes grouped by due date.
  2. Inbox Fix notes with missing tasks, invalid status, or vacation collisions.
  3. Hold Keep backlog notes visible without mixing them into today's work.
  4. Done Review completed tasks by date and context.

Screens

The three daily boards.

The documentation screenshots are generated by the executable acceptance suite in acceptance/specs/tasks-eye.e2e.ts. Each screen is captured in Obsidian Light, Obsidian Dark, and Dark with the Minimal theme.

Open

Open is the active board. It groups notes into No due, Today, Tomorrow, This Month, Next Month, and Future. Future work stays status: open; scheduling is represented by the Tasks due-date marker.

Inbox

Inbox is the repair queue. It surfaces open notes without an unfinished task, invalid frontmatter, closed notes with remaining work, and tasks scheduled on configured OOO days.

Hold

Hold is the backlog board. It uses the same compact row model as Open, but only for notes with status: hold.

Features

Small controls, explicit behavior.

Context filtering

Contexts come from folders below Db/. A note under Db/Mission/Allegro/ becomes M/Allegro. The toolbar filter narrows the board without changing note data.

Vacations and holidays

Tasks Eye marks custom OOO dates and configured holidays directly in the Open board. The OOO filter isolates those markers, while Inbox flags notes scheduled on unavailable days.

Uncheck selected tasks

The command Tasks Eye: Uncheck selected tasks works in the editor. It changes selected checked tasks back to unchecked and removes Tasks completion dates, using the Tasks API when it is available.

Done and daily summaries

The Done view collects completed Tasks items for a selected date. Daily notes can embed the same review with the ggajos-tasks-eye-daily-completed Markdown code block.

Executable documentation

Every feature above is backed by TypeScript.

The documentation is generated locally. The acceptance suite launches a sandboxed Obsidian app, installs the Tasks plugin and Minimal theme, opens a fresh fixture vault, exercises the plugin, and captures the screenshots used by this site. 

npm run acceptance:test
npm run docs:screenshots

To publish with GitHub Pages without CI, configure repository Pages to deploy from the /docs folder on the branch you choose.
