Obsidian plugin

Task boards that stay inside your notes.

Tasks Eye turns regular Markdown notes into focused boards, repair queues, review views, editor actions, and daily summaries. The feature documentation below is generated from executable feature folders.

Explore features Acceptance testing

Design intent

A quiet command center for note-centered work.

Tasks Eye is built for a vault where each piece of work lives as a note under Db/. The plugin reads note frontmatter, unfinished Tasks items, due dates, contexts from folders, vacation configuration, and completed-task markers. Feature folders now own the rationale, tests, screenshots, and generated documentation for that behavior.

Workflow

From active work to daily review.

  1. Open Review actionable notes grouped by due date.
  2. Inbox Fix one validation rule at a time.
  3. Hold Keep backlog notes visible outside today's work.
  4. Done Review completed tasks in a view or daily note.

Executable features

Every feature below comes from features/<slug>/.

Each folder provides typed metadata, why.md, feature-owned tests, and WDIO screenshots captured in Obsidian Light, Obsidian Dark, and Dark with the Minimal theme.

Board task controls

Board rows expose task actions for completing the next task and shifting due dates without opening the note.

The user can triage work directly from the board while Tasks Eye keeps the underlying Markdown task line as the source of truth.

Feature documentation

Uncheck selected tasks

The editor command turns selected completed task lines back into unchecked tasks and removes Tasks completion dates.

The user can reopen accidentally completed checklist items without manually cleaning Tasks emoji metadata.

Feature documentation

Vacation availability markers

Open boards show configured weekends, holidays, and custom OOO dates as availability markers through the work timeline.

The user can see unavailable dates next to scheduled work and isolate those dates with the synthetic OOO filter.

Feature documentation

Daily completed summary block

Daily notes can embed a completed-task summary with the ggajos-tasks-eye-daily-completed Markdown code block.

The user can review completed work inside the daily note where planning and reflection already happen.

Feature documentation

Vault conventions

Tasks Eye reads managed Markdown notes from Db/, timeline notes from Timeline/, status frontmatter, and Obsidian Tasks emoji metadata.

The user keeps a readable vault where the plugin adds views without owning the underlying data format.

Feature documentation

Context filtering

Tasks Eye derives context labels from Db/ folders and uses the toolbar filter to narrow board and Done views.

The user can focus on one area of life or work without changing note metadata or moving tasks between queues.

Feature documentation

Done view

The Done view collects completed Tasks items for a selected date and groups them by context and note.

The user can review what was completed on a specific day without searching individual notes.

Feature documentation

Hold

A backlog board for notes with status: hold, kept visible without mixing deferred work into the active Open board.

The user can keep future or paused work findable, reviewable, and filterable while preserving a clean active-work queue.

Feature documentation

Inbox view

Inbox is the repair queue that collects notes with validation violations, regardless of their current status.

The user gets one place to fix broken or inconsistent work notes before they pollute daily planning.

Feature documentation

Open view

Open is the active-work board for status: open notes, grouped by due-date buckets and ordered by next action.

The user can see actionable work by urgency while keeping future scheduling in normal Tasks due dates.

Feature documentation

Violation: closed note has unchecked tasks

A note with status: closed is invalid when it still contains unchecked task lines.

The user is warned when a supposedly finished note still contains unresolved work.

Feature documentation

Violation: invalid status

A note with explicit status outside open, hold, closed, or archived is invalid.

The user gets an immediate repair cue when a typo or obsolete status would otherwise hide a note from the intended workflow.

Feature documentation

Violation: open note has no uncompleted tasks

An open note is invalid when it has no unchecked task left to act on.

The user can close, archive, or add a next action instead of letting empty active notes clutter planning.

Feature documentation

Violation: task scheduled on vacation

An unfinished task with a due date on a configured unavailable day is reported as an Inbox violation.

The user can reschedule work before it lands on weekends, holidays, or custom OOO days.

Keyboard shortcuts

Default shortcuts for feature-backed commands.

These defaults are assigned by the plugin and can be changed in Obsidian's hotkey settings. Each shortcut links back to the feature that owns its behavior.

Default shortcut Command Feature Why
Mod+Shift+1 Open Tasks Eye: open Open view Jump to the active-work board.
Mod+Shift+2 Open Tasks Eye: inbox Inbox view Jump to the repair queue for validation violations.
Mod+Shift+3 Open Tasks Eye: hold Hold view Jump to backlog notes that should stay out of active work.
Mod+Shift+4 Open Tasks Eye Done Done view Open the completed-task review for the current day.
Mod+Shift+U Uncheck selected tasks Uncheck selected tasks Reopen selected checked task lines and remove completion dates.
